Включение режима unit_testing:
  • в корневой директории проекта сделать
    #make test all

Выключение режима unit_testing:
  • в корневой директории проекта сделать
    #make untest all

Добавление нового обьекта базы данных:
  1. в директории dbobjects создать соответствующие исходные файлы
  2. в процессе разработки создать тесты для данного обьекта базы данных в директории unit_tests
  3. унаследовать данный экземпляр объекта базы данных от dbobject_interface
  4. добавить определение соответствуюшей внутренней переменной Make файла в файл indDbobjects для определения включения/выключения компиляции исходных файлов (indHfiles indSfiles indOfiles) объекта базы данных и библиотек обьекта базы данных (indLfiles)
  5. добавить определение соответствующей переменной компилятора в файл dbobjects.h позволяющей включать/отключать включение (inclution) исходных файлов объекта базы данных в файлы src/main.cpp unit_tests/all_tests.cpp и создание при помощи оператора new соответствующего экземпляра обьекта базы данных в функции main файла src/main.cpp

Это позволит позднее (скажем, в случае отсутствия необходимости тестироваь данную БД или отсутствия в наличии самой БД) отключить использование данного объекта комментированием определений соотв двух переменных - в indDbobjects и dbobjects.h

Добавление нового теста базы данных:
  1. в директории еуыеыs создать соответствующие исходные файлы
  2. в процессе разработки создать тесты для данного теста базы данных в директории unit_tests
  3. унаследовать данный экземпляр объекта базы данных от test_interface
  4. добавить определение соответствуюшей внутренней переменной Make файла в файл indTests для определения включения/выключения компиляции исходных файлов (indHfiles indSfiles indOfiles) теста базы данных
  5. добавить определение соответствующей переменной компилятора в файл tests.h позволяющей включать/отключать включение (inclution) исходных файлов объекта базы данных в файлы src/main.cpp unit_tests/all_tests.cpp и создание при помощи оператора new соответствующего экземпляра теста базы данных в функции main файла src/main.cpp

Это позволит позднее (скажем, в случае отсутствия необходимости выполнять данный тест) отключить использование данного теста комментированием определений соотв двух переменных - в indTests и tests.h

По умолчанию программа вываливает результаты тестирования в html формате на консоль. Для сохранения (и понимания - все идет в UTF-8 $-) результатов нужно перенаправить вывод программы в файл.
